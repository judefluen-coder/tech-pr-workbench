## 发现的问题

### P1：yt-dlp 检测与实际执行路径不一致（已修复）

`/api/system/status` 报告 yt-dlp 未安装，但最近的抓取任务实际调用了 `/opt/homebrew/bin/yt-dlp`，随后搜索命令以非零状态退出并导致 YouTube 来源返回 502。

影响：用户可能看到“环境未安装”，同时后台又尝试调用另一个 yt-dlp；YouTube 抓取和下载的故障定位会变得不可信。

修复结果：新增共享运行时解析器，系统状态、YouTube 抓取和下载共用同一命令。状态接口返回 `/opt/homebrew/bin/yt-dlp (2026.07.04)`；真实 `ytsearch` 已返回视频、标题和封面。

### P2：未下载素材的提示文案不一致（已修复）

工作台按钮已经改为`下载视频`，但占位说明仍写“点击下载并翻译”。

影响：不会阻塞流程，但会让用户不确定点击后是否一定会执行翻译。

修复结果：统一为“下载视频并准备字幕”，按钮仍保持简短的“下载视频”。

### P2：导出弹窗仍然偏密集（已修复）

导出弹窗同时展示画幅、裁切、字幕、Logo、范围、版本、保存位置和完整预检。在 900px 高度下需要滚动才能看到最终按钮。

影响：主工作台已经清晰，但第三步进入弹窗后认知负担再次上升。

修复结果：默认只展示画幅、字幕、保存位置、文件名、风险摘要和导出按钮；裁切、Logo、范围、时长版本与完整预检折叠到“高级设置与导出检查”。

### P3：favicon 请求返回 404（已修复）

本地站点和 GitHub Pages 都出现一次 `favicon.ico` 404。没有 React/Vite 运行错误。

修复结果：本地 favicon 返回 HTTP 200，Playwright 控制台复测为 0 errors。
